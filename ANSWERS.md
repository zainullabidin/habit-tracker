## 2. Stack And Design Choices

I used Vanilla CSS, HTML and JS. The reason is as simple as the task - the task needed simple code, not some high level frameworks.

**Design decision 1:** I used a simple and charming colour theme which made it look pretty, warm, earthy, and calming so that it can soothe users. And some boxes with one-sided borders such as the QUOTE and the HABITS which are added as per user. It looked cool as in most of the sites (in past) I visited, they used cards which didn't feel good to me.

**Design decision 2:** I added a cartoonish character Mr. Habinator to make it seem interesting and enjoyable and it can be further improved by adding some cool animations to Mr. Habinator.

## 3. Responsive & Accessibility

On a narrow screen the layout doesn't adapt well. The text in the hero section gets squished and the sections don't stack on top of each other. The banner image scales fine tho. On a full laptop screen (1440px) everything looks clean with proper spacing.

**Accessibility handled:** All the images have alt text so screen readers can describe them.

**Accessibility skipped:** The major one can be the keyboard phase where you can freely move around the boxes of the table and add a note which appears only on pressing enter.

## 4. AI Usage

I used Claude (Anthropic) and Gemini (Google) throughout the project. Here is every instance:

- **CSS styling:** Asked Claude about flexbox properties (display flex, justify-content, align-items) and layout concepts. I learned padding vs margin, box-sizing, and how to style the grid table better through these conversations.

- **JavaScript logic:** Claude helped me understand addEventListener, localStorage, and how to build the weekly grid. I went through the code line by line, asking what each part does, comparing concepts to C++ which I already know (push = push_back, localStorage = fstream, querySelector = pointer to element).

- **Weekly grid rendering:** Claude provided the renderGrid function that builds the HTML table dynamically. I understood the logic - it's a nested for loop building a 2D table, same concept as printing a 2D array in C++.

- **Image Generation:** I used Gemini for creating images, described and edited every single image to align them with the content.

**What I changed:** Approximately everything! I tweaked each part generated by AI to meet my requirements and to learn new things.

## 5. Honest Gap

The biggest gap is the **responsive design**. On small screens it does not show up properly and gets squished. If I had more time I would've worked on it and most likely would have resolved it.

Another thing is the **rename habit feature**. For the moment we can only Add and Delete the records but we cannot alter them or rename them. That's a gap worth noticing.

The **Design Structure** could be improved by using Figma and adding proper connections and making it more visually appealing.

Also the **habit list styling** - it has some error, a minor one, but I couldn't get to it due to lack of time.

Overall this can be improved and made fully functional with some more effort, proper planning, and carrying out that plan effectively.
